# PingpongSmart v0.1.0 Checklist

> 每一项通过运行代码或观察行为来验证，聚焦系统行为。

## 实现完整性
- [ ] 后端启动类可正常启动 SpringBoot 应用（验证：`mvn spring-boot:run` 启动成功，无报错）
- [ ] `.env` 配置加载正确，LLM_BASE_URL、LLM_API_KEY、LLM_MODEL 可从环境变量读取（验证：启动时不打印密钥，通过断点或日志确认）
- [ ] `POST /api/chat` 接口存在并可被调用（验证：curl 请求返回 SSE 流）
- [ ] LlmService 通过 WebClient 正确调用大模型 API（验证：传入消息后收到模型回复）
- [ ] 前端页面能正常展示（验证：`npm run dev` 启动，浏览器打开无报错）
- [ ] 聊天消息以气泡形式展示（验证：用户消息右对齐蓝色，助手消息左对齐灰色）

## 交互行为
- [ ] 发送消息后输入框禁用，显示「思考中」状态（验证：点击发送后输入框变灰，显示加载提示）
- [ ] 模型回复完成后输入框恢复可用（验证：收到 done: true 后输入框可再次编辑）
- [ ] 流式响应逐字渲染（验证：回复逐字出现，非一次性展示）
- [ ] API 调用失败时前端展示错误提示（验证：断网或填写错误 API Key，显示错误消息）

## 编译与测试
- [ ] 后端 `mvn compile` 编译无错误
- [ ] 后端 `mvn test` 测试通过（如有单元测试）
- [ ] 前端 `npm install` 依赖安装无报错
- [ ] 前端 `npm run build` 构建无错误

## 集成
- [ ] 前后端同时运行，Vite proxy 正确转发 /api 到后端（验证：前端页面发请求，后端控制台收到日志）
- [ ] SSE 流式数据完整传输（验证：使用 Network 面板观察 SSE 事件流，确认有多条 data 事件）

## 端到端场景
- [ ] 场景 1：打开页面 → 输入「如何提高反手拧拉质量」→ 点击发送 → 等待流式回复完成 → 观察到逐字打字机效果 → 输入框恢复可用
- [ ] 场景 2：填写无效 API Key → 输入问题发送 → 页面展示错误提示（非技术堆栈）
- [ ] 场景 3：断网状态下发送 → 页面展示网络错误提示
- [ ] 场景 4：按 README.md 步骤从零开始 → 10 分钟内完成全部跑通
