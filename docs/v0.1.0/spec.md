# PingpongSmart v0.1.0 Spec

## 背景
PingpongSmart 定位是面向乒乓球用户的私有知识管理与智能问答平台。当前项目目录为空，从零起步。v0.1.0 作为最小可运行版本，目标是跑通「用户提问 → 大模型回答 → 流式展示」这条核心链路。

## 目标
- 用户在前端页面输入问题
- 后端接收问题，调第三方中转站（OpenAI 兼容 API）的大模型接口
- 以 SSE 流式将模型返回结果实时推送回前端
- 形成类似 ChatGPT 的聊天体验
- 支持通过 `.env.example` 配置 API Key 和 base_url

## 功能需求
- F1: 前端聊天页面 — 提供一个简洁的单页面应用，包含消息展示区和输入框，用户输入问题后点击发送，消息以对话气泡形式展示，流式接收后端返回的内容逐字渲染到页面，发送过程中显示「思考中」加载状态并禁用输入框
- F2: 后端聊天接口 — 提供 RESTful API（如 `POST /api/chat`），接收用户消息，调用指定的大模型接口（OpenAI 兼容格式），以 SSE 流式方式返回模型生成的响应
- F3: 大模型对接 — 支持通过配置文件（`.env`）指定 base_url、API Key、模型名称；默认 base_url 为 `https://maas-api.cn-huabei-1.xf-yun.com/v2`；支持流式请求（stream=true），将模型逐 token 转发给前端
- F4: 密钥配置 — 提供 `.env.example` 文件，列出所有必需配置项；后端通过环境变量读取配置，不在日志中打印密钥
- F5: README 文档 — 在 README.md 中写明初始化步骤：复制 `.env.example` 为 `.env`、填写 API Key、启动前后端命令

## 非功能需求
- N1: 首 token 响应时间（TTFT）控制在 3 秒以内（视网络情况）
- N2: API 调用失败时前端展示明确的错误提示；超时时间不少于 60 秒
- N3: 前端开发模式下支持 CORS，通过 Vite proxy 或后端跨域配置解决
- N4: 不引入用户系统、数据库、历史会话等复杂功能，保持 v0.1.0 尽可能轻

## 不做的事
- 用户注册/登录系统
- 聊天记录持久化（刷新页面后记录丢失）
- 乒乓球知识库 / RAG 检索（v0.1.0 仅做通用问答）
- 多轮对话上下文管理（每轮独立发送）
- 语音输入/输出
- 移动端适配（仅桌面端体验）

## 验收标准
- AC1: 前端页面能正常展示，包含消息展示区和输入框，用户可输入问题并发送
- AC2: 用户发送问题后，页面显示「思考中」状态，发送期间输入框禁用
- AC3: 后端接收请求后正确调用大模型接口（stream=true），以 SSE 格式返回流式响应
- AC4: 前端逐字渲染模型返回的内容，形成类 ChatGPT 的打字机效果
- AC5: 通过 `.env` 配置 base_url 和 API Key 后，能正常发起对话并获得有效回复
- AC6: API 调用失败时，前端展示错误提示信息（非技术堆栈）
- AC7: README.md 中清晰描述了初始化步骤（复制 .env.example → 填写密钥 → 启动命令）
- AC8: 项目整体可在 10 分钟内完成从零到跑通（拉代码 → 初始化 → 填密钥 → 启动 → 打开页面提问）
